Installation Guide

Option 1: Quick Start (If you have Python installed)

  1. Download the Test Grader script using the button above
  2. Open a terminal/command prompt in the download folder
  3. Run: python "test grader V6.0.0.py"

Option 2: Fresh Install with Python 3.14.0 & Virtual Environment

  1. Download and install Python 3.14.0 using the button above
  2. During installation, check "Add Python to PATH"
  3. Download the Test Grader script
  4. Open a terminal/command prompt in the script folder
  5. Create a virtual environment: 
    # Windows
python -m venv .venv
.venv\Scripts\activate
# use command prompt it is going to work.

# macOS/Linux
python3 -m venv .venv
source .venv/bin/activate
  6. Run the script: python "test grader V6.0.0.py"
  7. Enter the test score when prompted
  8. View detailed results with letter grade and feedback

Option 3: Professional Setup with Visual Studio Code

  1. Download and install Python 3.14.0
  2. Download and install Visual Studio Code
  3. Open VS Code and install the Python extension:
    • Click on Extensions icon (left sidebar) or press Ctrl+Shift+X
    • Search for "Python" and install the Microsoft Python extension
  4. Download the Test Grader script
  5. Open the script folder in VS Code: File > Open Folder
  6. Create a virtual environment using VS Code terminal: 
    # Open terminal in VS Code (Ctrl + `)
python -m venv .venv

# VS Code will prompt to select this as your interpreter - click Yes
# Or manually select: Ctrl+Shift+P > "Python: Select Interpreter" > Choose .venv
  7. VS Code will automatically activate the virtual environment
  8. Run the script by pressing F5 or clicking the Run button
  9. View the output in the integrated terminal

Why Use Visual Studio Code?

  • IntelliSense: Smart code completion and suggestions
  • Debugging: Built-in debugger with breakpoints and variable inspection
  • Integrated Terminal: Run Python scripts without leaving the editor
  • Extensions: Rich ecosystem of Python tools and linters
  • Git Integration: Version control built right in
  • Free & Open Source: Completely free to use

Why Use a Virtual Environment (.venv)?

  • Isolation: Keeps project dependencies separate from system Python
  • Clean: Prevents version conflicts between different projects
  • Portable: Easy to recreate the same environment on different machines
  • Professional: Industry best practice for Python development
